6. Flux du Chatbot — Question → Réponse

6.1 — Algorithme de matching

flowchart TD
    INPUT["Utilisateur tape :\noù manger sur le campus ?"]
    NORM["normalize\n1. Minuscules\n2. Supprime accents NFD\n3. Garde a-z 0-9 espaces\nResultat : ou manger sur le campus"]
    LOOP["Pour chaque entrée\nde BaseConnaissance\ncalcul du score"]
    MC["Mot-clé : manger\nnormalisé : manger\nmanger IN ou manger sur le campus\n→ +3 points"]
    MC2["Mot-clé : campus\nnormalisé : campus\ncampus IN ou manger sur le campus\n→ +3 points"]
    MC3["Mot-clé : restaurant\nNon trouvé → +0"]
    BEST["Meilleur score = 6\nMeilleur match = entrée 14"]
    CHECK{"Score >= 1 ?"}
    YES["Retourne entrée 14\navec sa réponse"]
    NO["Retourne None\n→ message par défaut"]

    INPUT --> NORM
    NORM --> LOOP
    LOOP --> MC
    LOOP --> MC2
    LOOP --> MC3
    MC --> BEST
    MC2 --> BEST
    MC3 --> BEST
    BEST --> CHECK
    CHECK -->|Oui| YES
    CHECK -->|Non| NO
Scoring : +3 points si le mot-clé complet est trouvé comme sous-chaîne dans le texte normalisé. +1 point par mot individuel du mot-clé trouvé dans les mots de l'input. Le seuil minimum est de 1 point.

6.2 — Flux complet requête ↔ réponse

sequenceDiagram
    actor E as Etudiant
    participant JS as Frontend JS
    participant API as api_send_message
    participant NLP as find_match
    participant DB as SQLite3

    E->>JS: Tape question + Envoyer
    JS->>API: POST /api/send/ JSON
    API->>API: Vérifie auth + CSRF + POST
    API->>API: Parse JSON
    API->>NLP: find_match(question)
    NLP->>DB: BaseConnaissance.objects.all()
    DB->>NLP: 30 entrées
    NLP->>NLP: Calcul des scores
    NLP->>API: Match trouvé ou None

    alt Match trouvé
        API->>DB: INSERT LogChatbot match=True
        API->>JS: JSON reponse + categorie + log_id
    else Aucun match
        API->>DB: INSERT LogChatbot match=False
        API->>JS: JSON message par défaut + log_id
    end

    JS->>E: Affiche réponse + boutons satisfaction
    E->>JS: Clique Utile
    JS->>API: POST /api/satisfaction/ log_id + oui
    API->>DB: UPDATE satisfaction = Utile
