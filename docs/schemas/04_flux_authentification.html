4. Flux d'Authentification et Contrôle d'Accès

4.1 — Inscription

sequenceDiagram
    actor U as Utilisateur
    participant B as Navigateur
    participant V as views.inscription()
    participant DB as SQLite3

    U->>B: Accède /register/
    B->>V: GET /register/
    V->>B: Formulaire InscriptionForm
    U->>B: Remplit username, email, mdp, rôle, formation
    B->>V: POST /register/ + CSRF token
    V->>V: form.is_valid()
    V->>DB: form.save() — INSERT Utilisateur (mdp hashé)
    V->>V: login(request, user) — crée session
    V->>B: redirect /dashboard/

4.2 — Connexion

sequenceDiagram
    actor U as Utilisateur
    participant B as Navigateur
    participant V as ConnexionView
    participant DB as SQLite3

    U->>B: Accède /login/
    B->>V: GET /login/
    V->>B: Formulaire ConnexionForm
    U->>B: Saisit username + password
    B->>V: POST /login/ + CSRF
    V->>DB: authenticate(username, password)
    DB->>V: OK — hash vérifié
    V->>V: get_success_url() — ?next= ou /dashboard/
    V->>B: redirect /dashboard/

4.3 — Chaîne de sécurité des vues

flowchart TD
    REQ["Requête entrante"] --> AUTH{"Utilisateur\nauthentifié ?"}
    AUTH -->|Non| REDIR_LOGIN["Redirect /login/?next=URL"]
    AUTH -->|Oui| ROLE{"@role_required\nprésent ?"}
    ROLE -->|Non| VUE_OK["Vue exécutée"]
    ROLE -->|Oui| CHECK{"user.is_superuser\nOU\nuser.role in roles ?"}
    CHECK -->|Oui| VUE_OK
    CHECK -->|Non| REDIR_DASH["Redirect /dashboard/"]
